# V20.0.0 SDK Sync

Mirror the V20 program split (30 → 40 instructions) in `packages/sdk`. Each
vault-aware builder now picks between the wallet-only and `_via_vault`
instruction at call time.

## D-1 — Branch on `vaultCreatorStr` per builder

Each SDK builder currently calls `program.methods.X()` once and passes vault
accounts as `null` when no vault. V20 program contexts no longer accept
Optional vault accounts, so the SDK must call the matching method
(`program.methods.X()` or `program.methods.XViaVault()`) and emit only the
accounts that belong to that context. No `null` vault keys in either path.

## D-2 — Public API surface unchanged

`BuyParams.vault?`, `DirectBuyParams`, `BorrowParams`, etc. keep their current
shape. Switching is internal. Consumers that already pass `vault` keep
working; agents using vault-funded flows transparently hit the new
`*_via_vault` ix; humans hit the wallet ix.

## D-3 — 10 builder sites

`buildBuyTransactionInternal`, `buildSellTransaction`,
`buildBorrowTransaction`, `buildRepayTransaction`,
`buildLiquidateTransaction`, `buildOpenShortTransaction`,
`buildCloseShortTransaction`, `buildLiquidateShortTransaction`,
`buildStarTransaction`, `buildClaimProtocolRewardsTransaction`. One branch
each; everything else (PDA derivation, ATA setup, memo, finalize) stays.

## D-4 — Remove `as any` casts on `.accounts({...})`

V19 needed the casts because Anchor's generated `IdlAccounts<T>` types
choked on the Optional vault triple. V20's clean non-Optional shapes mean
the cast is dead weight. Drop them in the same pass as D-3; if a cast is
still load-bearing for a different reason (e.g., unrelated IDL field name
divergence), keep it locally and note why inline.

## D-5 — Tests via public surface only

`tests/test_e2e.ts` and `tests/test_devnet_e2e.ts` call the public builders;
vault-using paths will exercise the new `_via_vault` ix without code change.
No new test scaffolding; out of scope to add per-instruction split tests.
